## Protótipo

O protótipo desenvolvido permite:

- Cadastro de usuários (nome e e-mail);
- Seleção de categorias de produtos de interesse;
- Escolha de faixas de preço por produto;
- Confirmação visual com resumo das preferências do usuário;
- Disparo automatizado de e-mails de confirmação (via Make/Integromat).

O funcionamento ocorre através de fluxos visuais configurados na plataforma escolhida, reduzindo a necessidade de código manual e agilizando testes e validações.

---

## Plataforma Utilizada

**Plataforma escolhida:** Make, cloude (Integromat)

### Justificativa da Escolha

A plataforma foi escolhida por oferecer:

- Interface visual intuitiva;
- Facilidade de integração com APIs e serviços externos, Vscode.
- Automação rápida de processos;
- Redução significativa do tempo de desenvolvimento.

Além disso, o Make possibilitou criar fluxos automatizados sem necessidade de programação avançada.

---

## ✅ Vantagens Identificadas

1. **Desenvolvimento rápido de protótipos** — Permitiu validar ideias em pouco tempo.
2. **Automação eficiente** — Processos repetitivos foram automatizados sem desenvolvimento complexo.
3. **Facilidade de manutenção** — Fluxos visuais tornam o sistema mais compreensível.
4. **Baixa barreira técnica** — Integrantes com pouca experiência em programação conseguiram participar.

---

## Limitações Encontradas

1. **Customização limitada** — Algumas funcionalidades dependiam dos recursos nativos da plataforma.
2. **Dependência da ferramenta** — O projeto ficou vinculado à infraestrutura da plataforma utilizada.
3. **Limitações de escalabilidade** — Recursos gratuitos possuem restrições de uso.
4. **Integrações pagas** — Alguns conectores avançados exigiam planos premium.

---

## Reflexão Crítica

Entendemos que plataformas Low Code/No Code aceleram significativamente a criação de soluções, especialmente em fases iniciais de prototipagem.

Para lidar com as limitações encontradas, foram propostas soluções como:

- Organização modular dos fluxos de automação;
- Planejamento de futura migração para soluções híbridas (Low Code + código tradicional).

A experiência demonstrou que essas ferramentas são excelentes para validação rápida de ideias e automação de processos simples e médios.
